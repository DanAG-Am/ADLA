¿Qué es el DOM?

El DOM (Document Object Model) es una interfaz de programación para documentos HTML y XML. Representa la estructura del documento como un árbol de nodos, donde cada nodo es un objeto que corresponde a una parte del documento.

Cita Textual

"Es una interfaz de programación para los documentos HTML y XML. Facilita una representación estructurada del documento y define de qué manera los programas pueden acceder, al fin de modificar, tanto su estructura, estilo y contenido. El DOM da una representación del documento como un grupo de nodos y objetos estructurados que tienen propiedades y métodos. Esencialmente, conecta las páginas web a scripts o lenguajes de programación." (Introducción - Referencia De La API Web | MDN, 2025)

¿Qué es un nodo?

Un nodo en el DOM es una unidad básica de la estructura del documento. Puede representar diferentes partes de un documento, como un elemento, un atributo o el contenido de texto.

Ejemplo:


    // Un nodo de tipo "elemento"
    var nodoElemento = document.getElementById("miElemento");

    // Un nodo de tipo "texto"
    var nodoTexto = document.createTextNode("Texto de ejemplo");

Dicho de manera simple, un nodo es una etiqueta HTML sobre la que vamos a poder realizar operaciones de lectura y de escritura." (Casero, 2025)

¿Cómo se accede a un elemento del DOM?

Se puede acceder a un elemento del DOM utilizando métodos como getElementById, getElementsByClassName o querySelector. Ejemplo: 


    // Obtener un elemento por su ID
    var elemento = document.getElementById("miElemento");

    // Obtener el primer elemento con una clase específica
    var primerElementoClase = document.querySelector(".miClase");

    // Obtener todos los elementos con una clase específica
    var elementosClase = document.getElementsByClassName("miClase");

¿Cómo se modifica un elemento del DOM?

Para modificar un elemento, se pueden usar propiedades como innerHTML o style. Ejemplo: 


    // Cambiar el contenido HTML de un elemento
    document.getElementById("miElemento").innerHTML = "Nuevo contenido";

    // Cambiar el color del texto de un elemento
    document.getElementById("miElemento").style.color = "red";

¿Cuál es la diferencia entre getElementBy* y querySelector*?

- getElementById() devuelve un solo elemento por su ID, mientras que querySelector() puede seleccionar elementos usando cualquier selector CSS.

- getElementsByClassName() devuelve una lista de nodos de los elementos con la clase dada, mientras que querySelectorAll() devuelve una lista de nodos de todos los elementos que coinciden con el selector CSS. 


    // getElementById
    var elemento = document.getElementById("miElemento");

    // querySelector (puede usar cualquier selector CSS)
    var elemento = document.querySelector("#miElemento"); // Selección por ID
    var elementos = document.querySelectorAll(".miClase"); // Selección por clase

¿Qué es un evento?

Un evento es una acción que ocurre en el navegador, como un clic del ratón, una pulsación de tecla, o el envío de un formulario.

Ejemplo de evento de clic:


    // Evento de clic en un botón
    document.getElementById("miBoton").addEventListener("click", function() {
        alert("¡Se hizo clic en el botón!");
    });

Cita Textual

"Los eventos son cosas que suceden en el sistema que estás programando. El sistema se encarga de producir una señal de cierto tipo cuando un evento ocurre, y proporciona un mecanismo para que una acción se lleve a cabo (ejecutar código) de forma automática cuando el evento ocurra. Los eventos se lanzan dentro de la ventana del navegador y usualmente están asociados a un elemento en específico dentro de dicha ventana. Esto puede ser un solo elemento, un grupo de elementos, el documento HTML cargado la pestaña actual, o la ventana del navegador en su totalidad. Existen distintos tipos de eventos que pueden ocurrir."(Introducción a Los Eventos - Aprende Desarrollo Web | MDN, 2025)

¿Qué es un listener?

Un listener es una función que se ejecuta cuando ocurre un evento específico en un elemento. Se asocia con el evento mediante el método addEventListener.

Ejemplo de listener para un evento de clic:


    // Listener para el evento de clic
    document.getElementById("miBoton").addEventListener("click", function() {
        alert("¡Hiciste clic en el botón!");
    });

Cita Textual

"El objeto que recibe una notificación cuando un evento de el tipo especificado ocurre. Debe ser un objeto implementando la interfaz EventListener o solo una function en JavaScript."(Introducción a Los Eventos - Aprende Desarrollo Web | MDN, 2025)

¿Qué es un callback?

Un callback es una función que se pasa como argumento a otra función, y se ejecuta después de que una operación ha finalizado, como una respuesta a un evento o un proceso asincrónico.

Ejemplo de callback:


    // Ejemplo de callback en una función asincrónica
    function procesarDatos(callback) {
        // Simulación de un proceso asincrónico
        setTimeout(function() {
            console.log("Datos procesados");
            callback(); // Llamada al callback después de procesar los datos
        }, 2000);
    }

    // Llamada a la función con un callback
    procesarDatos(function() {
        console.log("Callback ejecutado después de procesar los datos");
    });

Cita Textual

"Los callbacks (a veces denominados funciones de retrollamada o funciones callback) no son más que un tipo de funciones que se pasan por parámetro a otras funciones."(¿Qué Son Los Callbacks? - Javascript En Español, n.d.)

¿Qué es un objeto de evento?

Un objeto de evento contiene información sobre el evento que ha ocurrido, como el tipo de evento, el elemento sobre el que ocurrió y otros detalles específicos. Se pasa automáticamente a la función callback cuando un evento es disparado.

Ejemplo de uso de un objeto de evento:


    document.getElementById("miBoton").addEventListener("click", function(event) {
        // El objeto event contiene información sobre el evento
        console.log("Tipo de evento:", event.type); // Imprime el tipo de evento
        console.log("Elemento objetivo:", event.target); // Imprime el elemento que disparó el evento
    });

Cita Textual

"Los objetos de evento son estructuras JSON que se construyen automáticamente y se pasan como parámetros para activar funciones o devolver llamadas cuando un usuario interactúa con un complemento."(Objetos De Eventos, n.d.-b)

Además, la integración de IA en proyectos puede aumentar la complejidad y los costos iniciales.
