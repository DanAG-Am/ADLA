Workflow in Github and Git

Knowing the workflow enables you to track your projects and collaborate with others efficiently. Let's go through some github commands that take your remote repository to a local one and vice-versa.

Commands in Git:

  1. git init: Initialize a new repository.
  2. git clone [url]: Clone a repository locally.
  3. git status: Show the status of different files (if you added, deleted or committed any).
  4. git add [file]: Add files to the staging area
  5. git commit -m "message": Commit the files in the staging area, as well as add a message to describe to yourself or your team what the changes are about.
  6. git push: Upload your changes to the remote repository
  7. git pull: Bring the changes from the remote repository into the local one.
  8. git branch: Show any existing branches in your repository.

The same commands apply when working on a branch and pushing or pulling changes from it. Now, this is just a basic workflow. Lets take a look at how we can do different things in github to bring a repository to life.

Features in Github

Here are some of the features that you can use in github to make your workflow more appealing.

Features:

Github allows you to implement code, image from the repository or external sources, lists and different semantic features just like in html, such as headers.
You will mostly find such type of text in a README file, which describes the repository and the code implementation.

Github also allows you to track branches, check suggested workflow and the distribution of the programming languages found in your repository
